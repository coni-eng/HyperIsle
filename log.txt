1. ISSUE: Test Notification Stash Failure (Stash Loop)
   ANALYSIS:
   - Logs showed repeated 'IMMEDIATE_SNOOZE_OK' events for returning notifications.
   - The 'isReturningFromSnooze' check was failing because 'isSelfCancel' used 'sbn.key'.
   - 'sbn.key' can change between the original post and the re-post after snooze (especially on MIUI).
   
   SOLUTION:
   - Modified 'NotificationReaderService.kt' to use a stable composite key: 'packageName|id'.
   - Updated 'markSelfCancel' and 'isSelfCancel' to track stashed items using this stable key.
   - This ensures returning notifications are correctly identified and NOT re-snoozed.

2. ISSUE: Anchor Disappears on Telegram Notification (Real-world)
   ANALYSIS:
   - Logs showed 'ANCHOR_RESTORE' followed by 'ANCHOR_IDLE_SHOW', but the overlay was invisible.
   - The root cause was that 'dismissAllOverlays' removed the notification model but DID NOT reset the 'AnchorCoordinator' state.
   - The coordinator remained in 'NOTIF_EXPANDED' mode, but with no content to render, resulting in an empty/invisible window.
   
   SOLUTION:
   - Modified 'IslandOverlayService.kt' in 'dismissAllOverlays' and 'dismissNotificationOverlay'.
   - Added explicit calls to 'shrinkToAnchor(reason)' before 'showAnchorIdleOverlay()'.
   - This forces the coordinator state back to 'ANCHOR_IDLE', ensuring the pill is rendered.
